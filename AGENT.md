# cang (藏经阁) - AGENT.md

> 项目级规格文档 | 知识仓真相源 | SDD 初始化于 2026-03-30

## 项目概述

独立知识仓 + 蒸馏运行地 + 发布物产出地。所有正式知识以藏经阁为唯一长期归属。

**生态定位**: Huidu 知识真相源，为 V2 和侠客岛提供 L1-L4 知识资产

## 技术栈

| 层级 | 技术 | 说明 |
|------|------|------|
| 存储 | 文件系统 (Markdown, JSON, PDF) | 纯文本资产 |
| 蒸馏 | Kimi CLI (多模态) | PDF 提取与结构化 |
| 导出 | Python 脚本 | 消费者适配器 |
| 血缘 | JSON manifest | 溯源与版本管理 |

## 数据流

### 知识入仓流
```
PDF 文献 → raw/pdf/{category}/ → 分类蒸馏 → staging/ 
→ 审核 → 晋升 L1-L4 → lineage 记录血缘
```

### 发布物流
```
L1-L4 知识 → exporters/ → publish/current/consumers/{consumer}/
→ V2/侠客岛 只读消费
```

### 消费查询流
```
V2/侠客岛 请求 → 读取 publish/current/ → 返回结构化知识
```

## 关键设计决策

### 1. 知识真相源唯一性
- **原则**: 所有正式知识以藏经阁为唯一长期归属
- **执行**: V2 和侠客岛只消费 publish/，不反写

### 2. 发布物与真相源分离
- **真相源**: L1-L4 层级，持续更新
- **发布物**: publish/current/，版本快照
- **优势**: 消费方获得稳定视图，真相源可独立演进

### 3. L1-L4 分层模型
| 层级 | 内容 | 更新频率 |
|------|------|----------|
| L1 | 写作心法 | 低频 |
| L2 | 医学策略 | 中频 |
| L3 | 疾病通识 | 中频 |
| L4 | 产品证据 | 高频（新试验数据）|
